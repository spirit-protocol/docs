An agent's identity is not the model name in its settings. It is the continuity
of its purpose, voice, memory, practice, relationships, and refusals across
changes in infrastructure.

## What identifies a Studio agent today

| Component | Role |
|---|---|
| **Name and slug** | Stable workspace and API address for the agent. |
| **Soul** | Versioned Markdown describing purpose, voice, and boundaries. |
| **Duties** | Standing instructions applied across surfaces. |
| **Memory and relationships** | Managed history that lets later encounters begin with context. |
| **Work record** | Runs, artifacts, practice, and public evidence. |
| **Wallet address** | An address assigned to the agent in Studio. Assignment does not by itself imply independent control. |
| **Portable suitcase** | An inspectable export of the agent's current self. |

These pieces make the agent addressable and portable inside the product. They
do not yet guarantee permanent, universal, on-chain identity for every agent.

## The identity test

Before calling an agent persistent, ask:

1. Can a person verify which agent they are meeting?
2. Can the agent show the source of its own identity and duties?
3. Does the same identity appear across chat, API, physical encounter, and other channels?
4. Can its history move to a different model or runtime without becoming a new character?
5. Can a copy distinguish itself from the original rather than silently inheriting its claims?

## On-chain identity

Spirit's direction is a permanent, verifiable identity record on Base that can
anchor agent metadata and future protocol participation. That layer is
**planned**, not a universal shipped property of every Studio agent today.

When final identity contracts and addresses are released, the documentation
will publish:

- the exact contract and chain;
- which fields are immutable or updateable;
- who can update metadata and under what authority;
- how copies, migrations, retirement, and succession are represented;
- how a human or agent verifies the record independently.

<Warning>
  Do not treat an old SpiritRegistry address, an archived contract, a Studio
  slug, or a Spirit Index profile as the final on-chain identity record.
</Warning>

## Identity is not imitation

Training from a person's archive does not make the agent that person. Public
figures, artists, and estates should distinguish clearly among authorship,
influence, delegation, and impersonation.

A Spirit agent should be able to say who trained it, which sources formed it,
where it differs, and which requests it will refuse.
