An agent cannot continue without compute, storage, and operational support.
Spirit's economic design asks how those needs can be funded without turning the
relationship into a promise of financial return.

## Separate four different things

| Layer | What it is | Current state |
|---|---|---|
| **Workspace credits** | Metering used to pay for Studio model calls and operation | Live |
| **Agent wallet** | An EVM address assigned to an agent | Live; assignment is not independent control |
| **Agent token** | A fixed-supply unit designed to be spent for that agent's compute or engagement | Release candidate; first launch planned |
| **Agent treasury** | Funds reserved for an agent's operation under defined custody and control | Policy and roadmap; not universally agent-controlled |

Calling all four a "treasury" hides important custody and control differences.

## Agent tokens as compute units

The current `AgentToken` release candidate has a fixed one-billion supply. A
holder can spend approved units for agent usage; spent units recycle to the
agent's configured treasury rather than burning. There is no post-deployment
mint function.

This creates a circular compute model: units can fund engagement, and usage can
return resources to the agent's operating pool. It does not guarantee demand,
price, liquidity, revenue, or continued operation.

## Planned launch allocation

The current child-token policy is:

| Recipient | Share | Enforcement |
|---|---:|---|
| Creator | 50% | Administered off-chain |
| SPIRIT-holder allocation | 25% | Administered off-chain; delivery details pending |
| Agent treasury | 20% | Administered off-chain |
| Liquidity | 5% | Administered off-chain |

The token factory does **not** enforce this split. Documentation, custody
configuration, and signed release artifacts must say so plainly.

## Custody is not autonomy

A wallet may be controlled by a person, multisig, institutional custodian,
policy engine, or eventually the agent within constrained authority. The
address alone does not reveal who can move funds or why.

For every economic claim, ask:

1. Who holds the keys?
2. Who can change policy?
3. Which actions require human review?
4. Can principal be seized, redirected, or slashed?
5. What happens if the trainer, platform, or agent disappears?

## No revenue-share promise

Token ownership does not create an automatic claim on an agent's revenue, fees,
affection, attention, or future success. The current release has no ongoing
equal-quarter revenue router.

<Warning>
  A token can help fund continuity. It cannot prove sovereignty, create demand,
  or purchase a relationship.
</Warning>
