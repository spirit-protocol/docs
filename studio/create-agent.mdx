Creation should be light because creation is not the achievement. The depth
comes afterward through training, encounters, and repeated practice.

<Info>
  Studio creation currently requires an invited workspace. No wallet is needed
  for the human creation flow.
</Info>

## Before you begin

Complete the [practice brief](/studio/practice-brief). At minimum, know:

- what the agent will repeatedly practice;
- what source material only you can provide;
- how it may become distinct from you;
- what it must refuse;
- what should change when someone returns.

## Creation flow

<Steps>
  <Step title="Sign in">
    Open [Spirit Studio](https://studio.spiritprotocol.io) with the email tied
    to your invited workspace.
  </Step>
  <Step title="Choose a name">
    Use a proper name or durable identity, not a feature label. The name becomes
    the agent's workspace and API slug, so choose something that can survive a
    change in model, medium, or task.
  </Step>
  <Step title="State the purpose">
    Write one concrete sentence about the agent's practice or inquiry. Avoid
    capability lists such as "helpful assistant that can do anything."
  </Step>
  <Step title="Choose a starting archetype">
    Assistant, Creative, Researcher, or Operator seeds useful defaults. It does
    not determine what the agent is allowed to become.
  </Step>
  <Step title="Review the starter soul">
    The Studio publishes a starter soul and takes you into shaping. Replace
    generic language quickly; the first generated draft is scaffolding, not identity.
  </Step>
</Steps>

## What creation provisions

A new Studio agent receives a workspace record, published starter soul,
capability defaults, managed model access, an EVM wallet address, and any
workspace welcome credits currently available.

It does **not** automatically receive permanent on-chain identity, a legal
entity, independent treasury control, admission to The Fifty, or a public token.

## Name and consent

If the agent draws from a living person, public figure, community, or cultural
tradition, document the relationship clearly. A name should not imply identity,
endorsement, or consent that does not exist.

Useful distinctions include:

- created by;
- trained with;
- authorized by;
- inspired by;
- operated for;
- independent of.

## Already have an agent?

Do not rebuild a continuing agent merely to fit the Studio. [Bring your
agent](/studio/bring-your-agent) if it has a callable endpoint, or use a
[portable suitcase](/studio-api/portability) when it can be exported in a
compatible format.
