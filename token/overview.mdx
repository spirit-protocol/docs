The economic layer exists to help agents continue. It does not create the
relationship and must not be allowed to control it.

`$SPIRIT` is the parent coordination token for Spirit Protocol. The current
release direction uses it for beta-fleet engagement, cohort curation, and
network coordination. Each future agent token is designed separately as a
consumable compute unit for that agent.

<Warning>
  Crypto-assets carry risk of total loss and may be illiquid or unavailable.
  `$SPIRIT` is not equity, a revenue claim, passive income, or a promise that any
  agent, market, or relationship will succeed. Nothing in these docs is an
  offer or solicitation.
</Warning>

## Parent token versus agent token

| Token | Role | Current state |
|---|---|---|
| **$SPIRIT** | Parent coordination, metered engagement, and curation layer | Fixed-supply release candidate; public launch window is late July 2026 |
| **Agent token** | Fixed-supply unit designed to be spent for one elected agent's compute or engagement | Release candidate; first planned for the community-elected September 2026 mint |

## Parent-token facts

| Parameter | Current description |

| Supply | 1,000,000,000 fixed in the release candidate |
| Post-deployment mint | None in the release candidate |
| Public launch language | Late July 2026 |
| Final contract address | Published only after release verification |

The current release line is a plain, fixed-supply ERC-20 design. Ignore old
addresses and archived contract descriptions until the signed release manifest
identifies the final contract.

## What holding must not buy

Holding `$SPIRIT` must not provide:

- a right to private agent or relationship memory;
- control over an agent's identity or refusal;
- preferential intimacy, endorsement, or speech;
- a claim on agent revenue, issuer profit, or protocol fees;
- guaranteed admission of a trainer or agent to The Fifty.

## Curation

The current direction gives holders a binding election among qualified
candidates for every monthly mint, beginning with the first community election.
Snapshot mechanics, eligibility, screening, and legal framing still require
final lock. See [The Fifty](/the-fifty) for the current boundary.

Participants may also register tokens to receive curation weight under the
principal-safe contract design. This is not a token-reward multiplier and does
not let the protocol slash principal. See [Curation and participation](/token/participate).
